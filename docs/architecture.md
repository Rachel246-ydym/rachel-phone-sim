## 二、聊天模块

### 2.1 线下剧情模式（核心功能）

#### 2.1.1 长篇叙事
- AI 以第一/第三人称生成大段叙事文本
- 每段显示「第X段总结 · 累计至X条互动」
- 用户通过输入框发送行为/对话推动剧情
- **双击/长按内容弹出操作菜单**：
  - 重生成（重点）
  - 编辑（重点）
  - 删除（重点）
  - 引用（保留）
  - 多选（保留）

#### 2.1.2 剧情分支（IF功能）
- 同一故事中不同选择导致不同走向
- 在关键节点手动创建分支点
- 可从任意存档点开始新分支
- 每条分支有独立的故事线和记忆
- 自动行为（日记/朋友圈等）需绑定到某一条确定的 IF 线，不混用

#### 2.1.3 存档功能
- 故事任意节点可手动存档
- 每个存档包含：
  - 存档名称
  - 创建时间
  - **完整的剧情总结**（不截断）
  - 当前分支标识
- 存档支持：编辑、删除、加载（回到该存档点继续）
- 「新建剧情分支」按钮独立于设置保存，功能明确

#### 2.1.4 剧情设定页
- 外观主题切换（暗黑/白底/米白/深蓝）
- 默认加入角色记忆（开关）
- 文风描述（文本输入框，自定义叙事风格）
- 目标字数设置
- 每几条自动总结
- 叙事人称选择（第一人称/第三人称/混合视角）
- **所有设定修改后持久化保存**，刷新不丢失
- 保存时显示明确的成功提示

### 2.2 角色档案

| 字段 | 优先级 | 说明 |
|------|--------|------|
| 角色头像 | 重点 | 可上传图片 |
| 角色名称 | 重点 | 角色的名字 |
| 用户昵称 | 重点 | 角色怎么称呼用户 |
| 角色人设与背景 | 重点 | 大段文本框，外形/性格/背景故事 |
| 角色在线状态 | 重点 | 绿点显示「在线」 |

### 2.3 模型参数

| 参数 | 优先级 | 说明 |
|------|--------|------|
| 最少/最多回复条数 | 重点 | 控制 AI 单次回复的段落数 |
| 温度 Temperature | 重点 | 滑块控制 |
| 时间感知 | 重点 | 角色是否知道当前时间 |
| Top P | 重点 | 滑块控制 |
| 最大输出 Token 数 | 重点 | 数值输入 |
| 流式输出 | 重点 | 开关 |
| 上下文消息数量上限 | 重点 | 滑块，控制发送给 API 的历史消息数 |
| 语音设置 | 保留 | TTS 语音合成，后续添加 |
| 通话语音模式 | 保留 | 后续添加 |

### 2.4 记忆与回复

| 功能 | 优先级 | 说明 |
|------|--------|------|
| 记忆条数控制 | 重点 | 滑块 10~MAX，控制记忆上下文长度 |
| 回复模式 | 重点 | 手动回复 / 自动回复(可打断) / 自动回复(不打断) |

### 2.5 自动行为

| 功能 | 优先级 | 说明 |
|------|--------|------|
| 自动发送消息 | 重点 | 开关 + 间隔时间设置 |
| 自动生成日记 | 重点 | 开关 + 每日生成时间 + 绑定指定 IF 线 |
| 自动发送朋友圈 | 重点 | 开关 + 每日发送时间 + 绑定指定 IF 线 |

**关键约束**：自动行为生成的记忆必须绑定到用户指定的某一条 IF 线（剧情分支），不能将多条分支的记忆混在一起。

### 2.6 记忆核心

| 功能 | 优先级 | 说明 |
|------|--------|------|

| 按日期查看记忆 | 重点 | 日历视图 |

| 记忆标签筛选 | 重点 | 全部/自动总结/手动总结/手动添加/角色自剖/线下剧情/互动 |
| 记忆清理 | 重点 | 一键删除 + 按标签删除 |
| 记忆条目管理 | 重点 | 每条显示日期+内容，支持编辑/删除 |

| 功能 | 优先级 | 说明 |
|------|--------|------|

## 四、数据结构设计（IndexedDB）

### 4.1 Store 规划

| Store 名称 | 用途 | 关键字段 |
|-----------|------|---------|
| characters | 角色档案 | id, name, nickname, avatar, persona, modelParams, createdAt |
| messages | 聊天消息 | id, characterId, role, content, timestamp, storyBranchId? |
| stories | 线下剧情 | id, characterId, title, branchId, segments[], archives[] |
| storyBranches | 剧情分支 | id, storyId, parentBranchId, branchPoint, name |
| archives | 存档 | id, storyId, branchId, summary, snapshot, createdAt |
| memories | 记忆核心 | id, characterId, content, tag, source, createdAt |
| heartVoices | 心声记录 | id, characterId, content, createdAt |
| settings | 全局设置 | key-value 结构 |
| apiConfigs | API 配置 | id, url, key, model, isPrimary, usageStats |
| userProfile | 用户信息 | name, nickname, gender, age, aboutMe |

### 4.2 关键关系
- character → messages (1:N)
- character → stories (1:N)
- story → storyBranches (1:N，树状结构)
- story → archives (1:N)
- storyBranch → messages (1:N，分支隔离)
- character → memories (1:N)
- character → heartVoices (1:N)

## 五、开发顺序建议

### 第一阶段：基础骨架
1. Vite + React 19 + TS 项目初始化
2. IndexedDB storage service 封装
3. 全局状态 Context + useReducer 骨架
4. 手机外壳 UI（状态栏 + 底部导航 + 页面路由）
5. TypeScript 类型定义

### 第二阶段：聊天核心
6. 角色档案 CRUD
7. 聊天室基础 UI（消息列表 + 输入框）
8. DeepSeek API 调用封装 + 流式输出
9. 基础对话功能（发送 → AI 回复 → 显示）

### 第三阶段：线下剧情
10. 长篇叙事模式 UI + 交互（双击菜单）
11. 剧情分支系统（IF 线）
12. 存档功能（创建/加载/编辑/删除/总结）
13. 剧情设定页（文风/字数/总结频率/持久化保存）

### 第四阶段：记忆与智能
14. 记忆核心（自动总结 + 手动添加 + 标签筛选）
15. 情感连贯性（心声生成 + 显示）
16. 模型参数设置面板
17. 自动行为（消息/日记/朋友圈 + IF 线绑定）

### 第五阶段："我的"模块
18. API 设置（主/副 API、测试连接、用量账本）
19. 显示设置（全屏/换页模式）

### 第六阶段：体验完善
20. 聊天记录搜索（关键词 + 日期）
21. 关于我和TA（用户/角色信息卡片）
22. 重要时刻（核心记忆入口）
23. 主题色设置
